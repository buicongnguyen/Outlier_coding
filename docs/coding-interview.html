Study Goal

Sixteen of the coding questions that appear most often in software-engineering interviews (the kind collected on Glassdoor, LeetCode, and DSA question banks), each with a short problem statement, the key insight, a worked solution, and its time/space complexity. Solutions are in Python for readability - the patterns transfer to any language. Try each before opening it, and practice stating your approach and complexity out loud.

Pattern Cheat Sheet

Most interview problems reduce to one of a handful of patterns. Recognize the pattern and the solution usually follows.

Hash map

Trade space for O(1) average lookups: seen-before checks, counting, grouping, and complement search (Two Sum). Usually turns an O(n²) scan into O(n).

Two pointers

Walk a sorted array or list from both ends, or with a slow/fast pair. Solves pair-sum, palindrome, dedup, and cycle detection in O(1) extra space.

Sliding window

A moving [start, end] range over a sequence for "longest/shortest substring or subarray with a property." Expand end, shrink start to keep the window valid: O(n).

Stack / queue

Stack for matching pairs, nesting, and undo (Valid Parentheses); queue for FIFO and breadth-first level processing.

BFS / DFS

Traverse trees and graphs. BFS (queue) finds shortest unweighted paths and level order; DFS (recursion/stack) explores depth, connectivity, and backtracking.

Binary search

Halve a sorted search space each step for O(log n). Also "search the answer" for monotonic predicates (smallest x that works).

Dynamic programming

Break a problem into overlapping subproblems and cache results. Define the state, the recurrence, and the base case; fill a table bottom-up.

Complexity first

Always state Big-O time and space, and name the bottleneck. Interviewers care as much about the analysis as the working code.

1. Arrays and Strings

1. Two Sum EasyHash map

Problem: Given an array nums and a target, return the indices of the two numbers that add up to the target (exactly one solution; each element used once).

Approach: Store each value's index in a hash map as you go. For each x, check whether the complement target - x was already seen - that gives the pair in one pass.

def two_sum(nums, target):
    seen = {}                     # value -> index
    for i, x in enumerate(nums):
        if target - x in seen:
            return [seen[target - x], i]
        seen[x] = i
    return []

Time O(n)  ·  Space O(n). The brute-force double loop is O(n²); the hash map removes the inner scan.

2. Valid Parentheses EasyStack

Problem: Given a string of ()[]{}, decide whether every bracket is closed by the correct type in the correct order.

Approach: Push opening brackets on a stack; on a closing bracket, the top of the stack must be its matching open. The string is valid only if each close matches and the stack ends empty.

def is_valid(s):
    pairs = {')': '(', ']': '[', '}': '{'}
    stack = []
    for c in s:
        if c in '([{':
            stack.append(c)
        elif not stack or stack.pop() != pairs[c]:
            return False
    return not stack

Time O(n)  ·  Space O(n) for the stack.

3. Best Time to Buy and Sell Stock EasyRunning min

Problem: Given daily prices, find the maximum profit from a single buy then a later sell (0 if no profit is possible).

Approach: Track the minimum price seen so far; at each day the best sale today is price - min_so_far. Keep the max of those.

def max_profit(prices):
    min_price = float('inf')
    best = 0
    for p in prices:
        min_price = min(min_price, p)
        best = max(best, p - min_price)
    return best

Time O(n)  ·  Space O(1).

4. Maximum Subarray (Kadane's) MediumDP / greedy

Problem: Find the contiguous subarray with the largest sum and return that sum.

Approach: Kadane's algorithm. At each element, either extend the current subarray or start fresh at this element - whichever is larger. Track the best seen.

def max_subarray(nums):
    best = cur = nums[0]
    for x in nums[1:]:
        cur = max(x, cur + x)      # extend or restart
        best = max(best, cur)
    return best

Time O(n)  ·  Space O(1).

5. Group Anagrams MediumHash map

Problem: Group a list of words so that anagrams (same letters, any order) are in the same group.

Approach: Anagrams share a canonical form. Use the sorted letters (or a 26-count tuple) as a hash-map key and append each word to its bucket.

from collections import defaultdict

def group_anagrams(words):
    groups = defaultdict(list)
    for w in words:
        key = tuple(sorted(w))     # canonical form
        groups[key].append(w)
    return list(groups.values())

Time O(n · k log k) for n words of length k  ·  Space O(n · k). A count-tuple key drops the sort to O(n · k).

6. Longest Substring Without Repeating Characters MediumSliding window

Problem: Return the length of the longest substring with all distinct characters.

Approach: Slide a window [start, i]. Remember each character's last index; when the current character repeats inside the window, jump start just past its previous occurrence. The window is always duplicate-free.

def length_of_longest_substring(s):
    last = {}                      # char -> last index
    start = best = 0
    for i, c in enumerate(s):
        if c in last and last[c] >= start:
            start = last[c] + 1
        last[c] = i
        best = max(best, i - start + 1)
    return best

Time O(n)  ·  Space O(min(n, alphabet)).

2. Linked Lists

Assume class ListNode: def __init__(self, val=0, next=None): self.val, self.next = val, next.

7. Reverse a Linked List EasyPointers

Problem: Reverse a singly linked list and return the new head.

Approach: Walk the list once, flipping each node's next to point at the previous node. Keep three pointers: previous, current (head), and the saved next.

def reverse_list(head):
    prev = None
    while head:
        nxt = head.next            # save next
        head.next = prev           # flip pointer
        prev = head                # advance prev
        head = nxt                 # advance head
    return prev                    # new head

Time O(n)  ·  Space O(1) iterative (recursion would be O(n) stack).

8. Merge Two Sorted Lists EasyTwo pointers

Problem: Merge two sorted linked lists into one sorted list.

Approach: Use a dummy head to simplify edge cases. Repeatedly attach the smaller of the two front nodes; when one list runs out, attach the remainder of the other.

def merge(a, b):
    dummy = tail = ListNode(0)
    while a and b:
        if a.val <= b.val:
            tail.next, a = a, a.next
        else:
            tail.next, b = b, b.next
        tail = tail.next
    tail.next = a or b             # attach the rest
    return dummy.next

Time O(n + m)  ·  Space O(1) (relinks existing nodes).

9. Detect a Cycle (Floyd's) EasyFast/slow pointers

Problem: Determine whether a linked list contains a cycle.

Approach: Floyd's tortoise and hare. Advance a slow pointer one step and a fast pointer two steps. If there is a cycle they eventually meet; if fast reaches the end, there is none.

def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False

Time O(n)  ·  Space O(1) - beats the O(n)-space hash-set approach.

3. Trees and Graphs

Assume a binary-tree node with val, left, and right.

10. Maximum Depth of a Binary Tree EasyDFS recursion

Problem: Return the number of nodes on the longest root-to-leaf path.

Approach: The depth of a tree is one plus the deeper of its two subtrees. Recurse; an empty subtree has depth 0.

def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))

Time O(n)  ·  Space O(h) recursion stack (h = tree height).

11. Validate a Binary Search Tree MediumDFS + bounds

Problem: Check whether a binary tree is a valid BST (every left descendant < node < every right descendant).

Approach: Carry a valid open range (lo, hi) down the tree. Each node must lie strictly inside it; going left tightens the upper bound to the node's value, going right tightens the lower bound. A common bug is only comparing a node with its direct children - the range fixes that.

def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root:
        return True
    if not (lo < root.val < hi):
        return False
    return (is_valid_bst(root.left,  lo, root.val) and
            is_valid_bst(root.right, root.val, hi))

Time O(n)  ·  Space O(h).

12. Binary Tree Level-Order Traversal MediumBFS queue

Problem: Return the node values level by level, top to bottom, as a list of lists.

Approach: Breadth-first search with a queue. Before draining the queue for a level, record its current length so you process exactly that level, collecting each node's children for the next.

from collections import deque

def level_order(root):
    if not root:
        return []
    res, q = [], deque([root])
    while q:
        level = []
        for _ in range(len(q)):    # one full level
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        res.append(level)
    return res

Time O(n)  ·  Space O(n) for the queue and output.

13. Number of Islands MediumGrid DFS/BFS

Problem: Given a grid of '1' (land) and '0' (water), count the connected islands (4-directional).

Approach: Scan the grid; each unvisited land cell starts a new island. Flood-fill it (DFS or BFS), sinking visited land to '0' so it is not counted again.

def num_islands(grid):
    if not grid:
        return 0
    rows, cols = len(grid), len(grid[0])

    def sink(r, c):
        if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] != '1':
            return
        grid[r][c] = '0'
        sink(r+1, c); sink(r-1, c); sink(r, c+1); sink(r, c-1)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                sink(r, c)
    return count

Time O(rows · cols)  ·  Space O(rows · cols) worst case (recursion / queue).

4. Searching and Dynamic Programming

14. Binary Search EasyBinary search

Problem: Return the index of target in a sorted array, or -1 if absent.

Approach: Keep a [lo, hi] range and compare the middle element to the target, discarding the half that cannot contain it each step. Use lo + (hi - lo) // 2 to avoid overflow in languages where it matters.

def binary_search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

Time O(log n)  ·  Space O(1).

15. Climbing Stairs EasyDP (Fibonacci)

Problem: You can climb 1 or 2 steps at a time. How many distinct ways to reach the top of n stairs?

Approach: Ways(n) = Ways(n-1) + Ways(n-2): the last move is either one or two steps. That is Fibonacci; roll two variables instead of a full table for O(1) space.

def climb_stairs(n):
    a, b = 1, 1                    # ways to reach step 0 and 1
    for _ in range(n):
        a, b = b, a + b
    return a

Time O(n)  ·  Space O(1).

16. Coin Change (fewest coins) MediumDP (unbounded)

Problem: Given coin denominations and an amount, return the fewest coins that sum to it, or -1 if impossible.

Approach: Bottom-up DP. dp[a] = fewest coins to make amount a. For each amount, try every coin: dp[a] = min(dp[a], dp[a - coin] + 1). Start dp[0] = 0 and the rest at infinity.

def coin_change(coins, amount):
    dp = [0] + [float('inf')] * amount
    for a in range(1, amount + 1):
        for c in coins:
            if c <= a:
                dp[a] = min(dp[a], dp[a - c] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

Time O(amount · coins)  ·  Space O(amount). Greedy (largest coin first) is not correct for arbitrary denominations - DP is.

How to Answer in the Interview

A strong answer is a small script: clarify inputs and edge cases (empty, single element, duplicates, negatives), state the approach and its Big-O before coding, write clean code, then walk one example through it and mention the trade-off you made (usually time vs. space). Naming the pattern - "this is a sliding window" - signals that you recognize the shape, which is what interviewers are really testing.
