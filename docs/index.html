Reading & References

Free, authoritative sources to read alongside each study area - books, interactive tools, and reference documentation. Everything here is readable online at no cost.

Core skill

Coding & Debugging

The fastest way to internalize data structures is to see them move: VisuAlgo animates sorting, trees, heaps, hashing, and graphs step by step, while the Big-O Cheat Sheet collects the time/space complexities of common structures and sorts in one table worth memorizing. For depth, MIT's 6.006 (OpenCourseWare) teaches the reasoning behind the complexities - invariants, amortized analysis, why hashing works. On the debugging side, the git-bisect documentation is the canonical description of binary-searching history for the commit that broke you, and freeCodeCamp's DSA guide is a gentle structured pass over the whole area with exercises.

Systems knowledge

Compilers & Porting

Crafting Interpreters is the standout free book here: you build a complete language twice - first a tree-walk interpreter in Java, then a bytecode VM in C - and every pipeline stage on the study page (lexer, parser, AST, codegen) becomes something you have personally written. After it, the LLVM Kaleidoscope tutorial shows how a production compiler frontend feeds a real optimizing backend, with "How to learn compilers: LLVM Edition" as a map of what to read in which order. For the porting half, Martin Fowler's strangler-fig article is the original description of incremental migration, and hyrumslaw.com is the one-page statement of why observable behavior - not documentation - is your real API contract.

Crafting Interpreters (free book) LLVM Kaleidoscope tutorial How to learn compilers: LLVM Edition Fowler: Strangler Fig Application Hyrum's Law
Language deep dive

Expert C

Jens Gustedt's Modern C (free from the author) is the current-standard deep read - it teaches C17/C23 idioms rather than 1989 habits. Beej's Guide to C is the friendlier companion volume, good for filling gaps quickly. Day to day, cppreference.com is the accurate language and library reference (far more reliable than most tutorial sites), and the comp.lang.c FAQ answers the classic traps - pointers vs arrays, sequence points, undefined behavior - that assessments love. The SEI CERT C Coding Standard documents, rule by rule, how C code goes wrong in the field and what secure code does instead.

Modern C (Gustedt, free) Beej's Guide to C cppreference (C) comp.lang.c FAQ SEI CERT C Standard
Version control

Git Mastery

Pro Git (the official book, free on git-scm.com) is the definitive text - chapters 2-3 and 7 cover 90% of daily work, and chapter 10 ("Git Internals") explains the object model that makes everything else click. Learn Git Branching is an interactive sandbox that visualizes commits, branches, rebases, and cherry-picks as you type the commands - the best way to build intuition safely. Atlassian's tutorials shine on workflow comparisons (merge vs rebase, trunk vs git-flow), and Oh Shit, Git! is the honest quick-reference for recovering from the mistakes everyone eventually makes.

Practice format

Why MCQ Drills Work

The format of these pages is deliberate. Cognitive psychology calls it the testing effect (retrieval practice): actively pulling an answer from memory strengthens retention far more than re-reading the same material. In the landmark Roediger & Karpicke experiments, students who tested themselves retained roughly 80% of a passage after a week versus about 34% for students who only re-read it. Explanations after each answer matter too - immediate feedback corrects misconceptions while they are fresh. For best results, return to these pages over spaced intervals (days apart) rather than in one sitting: spacing plus retrieval is the most robust combination the learning-science literature has produced.
